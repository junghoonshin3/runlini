# Ghost Run TTS Dispatcher Plan

## 목표

고스트런 중 경로 이탈처럼 중요한 음성 안내가 기존 TTS 재생 중에 묻히지 않도록 음성 안내 큐에 우선순위를 추가하고, 긴급 안내는 현재 음성을 중단한 뒤 즉시 재생한다.

## 가정

- 사용자 테스트에서 문제는 출발 확인 이후, 러닝 중 경로 이탈 안내가 나와야 하는 상황에서 발생했다.
- 출발지 도착지 판정과 플래그 표시는 이번 수정 범위가 아니다.
- Android `flutter_tts`의 `speak` 반환값이 `0`일 때는 안내가 실제로 수락되지 않은 것으로 본다.
- 긴급 고스트 안내는 현재 재생 중인 거리 안내보다 우선한다.

## 구현 계획

1. `RunVoiceCueClient.speak`가 실제 발화 수락 여부를 `bool`로 반환하도록 바꾼다.
2. `RunVoiceCue`에 우선순위를 추가하고 고스트 경로 이탈, 복귀, 완료 안내를 긴급으로 분류한다.
3. `RunVoiceCueDispatcher`를 추가해 일반 안내는 직렬 재생하고 긴급 안내는 현재 음성을 중단한 뒤 재생한다.
4. `RunningTabScreen`에서 직접 TTS를 호출하지 않고 dispatcher를 통해 안내를 보낸다.
5. 관련 fake와 테스트를 갱신하고 focused test, guardrails, analyze, flutter test를 실행한다.

## 검증 기준

- 일반 안내 재생 중 긴급 안내가 들어오면 `stop`이 호출되고 긴급 안내가 재생된다.
- 일반 안내 재생 중 낮은 우선순위 안내가 들어오면 끼어들지 않는다.
- Android TTS가 `0`을 반환하면 실패로 기록되어 dispatcher가 인지할 수 있다.
- 기존 고스트 이벤트 우선순위 테스트가 통과한다.
