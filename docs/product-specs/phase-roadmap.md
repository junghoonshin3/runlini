# Phase Roadmap

See also:

- `runlini-feature-priorities.md` for the concrete near-term product stack
- `later-features-backlog.md` for deferred features such as real-time location sharing

## Phase 1

- geolocator-backed recording
- sqflite storage
- route display with Google Maps on Android and Apple Maps on iOS

## Phase 2

- ghost selection
- time-based interpolation
- live gap feedback and haptics

## Phase 3

- `health` import flow
- Health Connect and HealthKit permission handling
- route merge policy

## Phase 4

- pace-colored route segments
- report screen
- full neo-brutalist finish
